## 📋 前置要求

开始之前，请确保您的系统已安装以下软件：

### 必需软件
- **Node.js 18+** - [下载 Node.js](https://nodejs.org/)
- **pnpm** - 包管理器 (`npm install -g pnpm`)
- **PostgreSQL** - 数据库托管商([Neon](https://neon.tech/))或本地数据库服务器
- **Git** - 版本控制系统 ([下载 Git](https://git-scm.com/))

### 可选服务（获得完整功能）
- **Stripe 账户** - 用于支付处理 ([注册](https://stripe.com/))
- **Cloudflare R2** 或 **AWS S3** - 用于文件存储
- **GitHub OAuth 应用** - 用于社交登录
- **Google OAuth 应用** - 用于社交登录

## ⚡ 5分钟设置

### 步骤 1：克隆仓库

```bash
# 克隆仓库
git clone https://github.com/justnode/better-saas
cd better-saas

# 或使用 GitHub CLI
gh repo clone https://github.com/justnode/better-saas
cd better-saas
```

### 步骤 2：安装依赖

```bash
# 安装所有依赖
pnpm install

# 这将安装：
# - Next.js 15 和 React 18
# - TypeScript 和所有类型定义
# - Tailwind CSS 和 UI 组件
# - 数据库工具 (Drizzle ORM)
# - 测试框架 (Jest, Playwright)
# - 以及所有其他项目依赖
```

### 步骤 3：环境配置

```bash
# 复制环境变量模板
cp env.example .env

# 在编辑器中打开 .env 文件
code .env  # VS Code
# 或
vim .env  # 终端编辑器
```

**配置基本的环境变量：**

```bash
# 应用配置
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# 数据库（必需）
DATABASE_URL="postgresql://username:password@localhost:5432/better_saas"

# 身份认证（必需）
BETTER_AUTH_SECRET="your-super-secret-key-here"

# 管理员配置
ADMIN_EMAILS="admin@example.com"
```

### 步骤 4：数据库设置
登录Neon，创建一个新项目，在项目面板中点击"Connect",找到数据库URL，将数据库URL添加到 .env 文件中。

```bash
# 将Neon中的数据库URL添加到 .env 文件中
DATABASE_URL="your-neon-database-url"

# 推送数据库架构
pnpm db:push

# 这将：
# - 创建所有必要的表
# - 初始化数据库结构
```

### 步骤 5：启动开发服务器

```bash
# 启动开发服务器
pnpm dev

# 应用程序将在以下地址可用：
# http://localhost:3000
```

🎉 **恭喜！** 您的 Better SaaS 应用程序现在正在运行！

## 🔧 完整配置

为了获得完整功能，请配置这些额外的服务：

### 支付集成（Stripe）

1. **在 [stripe.com](https://stripe.com/) 创建 Stripe 账户**
2. **从 Stripe 仪表板获取您的 API 密钥**
3. **添加到您的 `.env` 文件：**

```bash
# Stripe 配置
STRIPE_SECRET_KEY="sk_test_your_stripe_secret_key"
STRIPE_WEBHOOK_SECRET="whsec_your_webhook_secret"
NEXT_PUBLIC_STRIPE_PRICE_PRO_MONTHLY="price_monthly_id"
NEXT_PUBLIC_STRIPE_PRICE_PRO_YEARLY="price_yearly_id"
```

4. **在您的 Stripe 仪表板中创建产品和价格**
5. **为支付事件设置 webhook 端点**

### 文件存储（Cloudflare R2）

1. **创建 Cloudflare 账户并设置 R2**
2. **创建存储桶**
3. **添加到您的 `.env` 文件：**

```bash
# Cloudflare R2 配置
R2_BUCKET_NAME="your-bucket-name"
R2_ACCESS_KEY_ID="your-access-key-id"
R2_SECRET_ACCESS_KEY="your-secret-access-key"
R2_ENDPOINT="https://your-account-id.r2.cloudflarestorage.com"
R2_PUBLIC_URL="https://your-domain.com"
```

### 社交登录

#### GitHub OAuth
1. **创建 GitHub OAuth 应用：**
   - 转到 GitHub 设置 → 开发者设置 → OAuth 应用
   - 创建新的 OAuth 应用
   - 设置授权回调 URL 为：`http://localhost:3000/api/auth/callback/github`

2. **添加到您的 `.env` 文件：**

```bash
# GitHub OAuth
GITHUB_CLIENT_ID="your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"
```

#### Google OAuth

1. **创建 Google OAuth 应用：**
   - 转到 [Google Cloud Console](https://console.cloud.google.com/)
   - 创建新项目或选择现有项目
   - 启用 Google+ API
   - 创建 OAuth 2.0 凭据
   - 设置重定向 URI 为：`http://localhost:3000/api/auth/callback/google`

2. **添加到您的 `.env` 文件：**

```bash
# Google OAuth
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
```

## 🛠️ 开发命令

以下是开发的基本命令：

### 开发服务器
```bash
# 启动带热重载的开发服务器
pnpm dev

# 使用特定端口启动
pnpm dev -- --port 3001

# 使用 turbo 模式启动（更快的构建）
pnpm dev --turbo
```

### 数据库操作
```bash
# 将架构更改推送到数据库
pnpm db:push

# 生成数据库迁移
pnpm db:generate

# 运行数据库迁移
pnpm db:migrate

# 打开数据库工作室（GUI）
pnpm db:studio
```

### 代码质量
```bash
# 运行 TypeScript 类型检查
pnpm typecheck

# 运行代码格式化和检查
pnpm check

# 修复代码格式问题
pnpm check:write
```

### 测试
```bash
# 运行所有测试
pnpm test

# 仅运行单元测试
pnpm test:unit

# 运行集成测试
pnpm test:integration

# 运行端到端测试
pnpm test:e2e

# 运行带覆盖率的测试
pnpm test:coverage
```

### 构建和生产
```bash
# 为生产构建
pnpm build

# 启动生产服务器
pnpm start

# 构建并启动（组合）
pnpm preview
```

## 🚨 常见问题与解决方案

### 环境变量未加载

**问题：** 环境变量未定义
```bash
# 确保 .env 文件在根目录
ls -la .env

# 检查文件权限
chmod 644 .env

# 重启开发服务器
pnpm dev
```

### 端口已被占用

**问题：** `端口 3000 已被占用`
```bash
# 查找并终止使用端口 3000 的进程
lsof -ti:3000 | xargs kill -9

# 或使用不同端口
pnpm dev -- --port 3001
```

### Node.js 版本问题

**问题：** `不支持的 Node.js 版本`
```bash
# 检查您的 Node.js 版本
node --version

# 更新到 Node.js 18+
nvm install 18
nvm use 18
```
